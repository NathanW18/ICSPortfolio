Let‚Äôs say we wanted to store the final marks of a class of twenty students. We could make twenty int variables, each corresponding to one student‚Äôs marks. But that would be very inefficient to create and very challenging to compare and update. Instead, we use arrays.

Since arrays are a ‚Äúcollection‚Äù the same variable, they are declared with the variable data type.

int[] marks = new int[20]; // creates integer array with 20 elements
String[] names = new String[20]; // creates String array with 20 elements

If we wanted to initialize an array, either after or directly with declaration, we would do this.

int[] marks = new int[7];
marks = {100, 90, 56, 93, 92, 95, 87};
// or //
int[] marks = {100, 90, 56, 93, 92, 95, 87};

If we wanted to access an element of an array, we first need to find the index of the element. In Java arrays, the index always starts at 0. This means the first element of the array has index 0, the second element has index 1, the third element has index 2, and so on.

Interestingly, the arrays an the first code snippet above with marks and names are parallel arrays. This means that for each index of the two parallel arrays, they refer to the same body (one student). For example, the first index of marks refers to the mark of the first student, while the first index of the second array refers to the name of the first student.

We can find the index of the element, either by manually counting, or through looping (iterating) through the array using a for loop.

Common Mistakes

  1. Forgetting array indices begin at 0
    1. Result: All of your indices will be one off, resulting in an ArrayIndexOutOfBounds exception when you attempt to retrieve the last element of the array
  2. Using .length as a method (.length())
    1. Result: There is no such method called .length(), length is an attribute, not a method
  3. For 2D arrays: swapping the row and column indices
    1. Result: You either get the wrong value OR get an ArrayIndexOutOfBounds exception

Array Applications

  1. If you want to store a list of all the same elements, or keep several lists of attributes that pertain to the same object (i.e., parallel arrays)
  2. Storing large amounts of data and retrieving them through loops
  3. Sorting, changing, and accessing a particular piece of data
